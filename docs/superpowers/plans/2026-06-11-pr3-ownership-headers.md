# PR 3: Ownership Headers + Overwrite Gate — Implementation Plan

**Goal:** Establish the file-level ownership contract: tool-owned files carry a
`// Code generated by xp-provider-gen. DO NOT EDIT.` header and may be overwritten;
user-owned files have no header and are never touched. Provide the overwrite-gate
decision logic (used by `update` in PR5).

**Decisions resolved:** `setup.go` is **tool-owned** (header; drift detection will
protect hand-edits in PR5). `crossplane.yaml` is **seed-once** (no header → never
re-touched), as are the Makefile/Dockerfile/go.mod/README and the `*_types.go`/
`controller.go` user files.

**Architecture:**
- `core/ownership.go`: `GeneratedHeader` constant, `IsToolOwned(content) bool`
  (scans the file head for the header), `DecideWrite(exists, existing) WriteDecision`
  (Seed if absent, Overwrite if tool-owned, Skip otherwise).
- Add the header (after `{{ .Boilerplate }}`) to the 8 tool-owned Go templates and
  the 2 register generators. Leave `controller.go.tmpl`, `*_types.go.tmpl` headerless.

**Ownership map:**
- Tool-owned (header): `doc.go`, `generate.go`, `groupversion_info.go`,
  `apis/v1alpha1/register.go`, `cmd/provider/main.go`, `config.go`, `setup.go`,
  `version.go`, and both generated register files.
- User-owned (no header): `controller.go`, `<kind>_types.go`, `apis/v1alpha1/types.go`.

### Tasks

2. Implement `core/ownership.go`.

- The gate (`DecideWrite`) is exercised by the `update` command in PR5; here it ships
  with unit tests as the foundation.
